Hugo基础

zzl 18 October, 2019 share

1.Hugo的优点

  • 静态文件，安全、备份方便；
  • 所有页面数据源头都是一个md文件，纯文本（用什么软件都可以打开）、格式统一、管理方便；
  • 安装方便，不需要依赖复杂的运行环境；

2.安装Hugo

  1. 下载二进制安装包，地址：https://github.com/gohugoio/hugo/releases
  2. 创建一个新文件夹：C:\Hugo，你可以在任何地方创建，这里将Hugo放在C盘；
  3. 在Hugo文件夹中创建一个子文件夹： C:\Hugo\bin；
  4. 在Hugo中创建另一个子文件夹： C:\Hugo\Sites；
  5. 将下载的安装包内容解压到C:\Hugo\bin；
  6. 将C:\Hugo\bin添加到你的环境变量Path;

3.创建一个基础站点

  1. 进入C:\Hugo\Sites，运行hugo new site 你的项目名称；

    PS C:\Hugo\Sites> hugo new site hugoShare
Congratulations! Your new Hugo site is created in C:\Hugo\Sites\hugoShare.
   
Just a few more steps and you're ready to go:
   
1. Download a theme into the same-named folder.
  Choose a theme from https://themes.gohugo.io/ or
  create your own with the "hugo new theme <THEMENAME>" command.
2. Perhaps you want to add some content. You can add single files
  with "hugo new <SECTIONNAME>\<FILENAME>.<FORMAT>".
3. Start the built-in live server via "hugo server".

    命令执行成功之后查看C:\Hugo\Sites\hugoShare目录，查看文件目录结构

    │  config.toml
│
├─archetypes
│      default.md
│
├─content
├─data
├─layouts
├─static
└─themes

    • config.toml
      站点全局的参数配置文件

    • archetypes
      Hugo的markdown文件中前置数据Front Matter定义的结构，默认使用的是default.md文件，可以自定义，然后在config.toml中指定自定义的结构文件，打开default.md文件。

      ---
title: "{{ replace .Name "-" " " | title }}"
date: {{ .Date }}
draft: true
---

      这是一种yaml格式，Front Matter支持三种格式，除了yaml，还支持toml，json方式。

    • content
      存放网页内容的目录，即我们编写的markdown文件都存放在此目录，此目录是Hugo的默认源目录。

    • data
      data目录用来存放数据文件，一般是json文件，Hugo提供了相关命令可以从data目录下读取相关的文件数据，然后渲染到HTML页面中，将业务数据与模板分离。

    • layouts
      存放自定义的模板文件，Hugo优先使用layouts目录下的模板，未发现再去themes目录下查找。

    • static
      存放静态文件，比如css、js、img等文件目录，Hugo在渲染时，会直接将static目录下的文件直接复制到public目录下，不会做任何渲染。

    • themes
      存放网站主题，可以下载多个主题，themes目录下的每个子目录代表了一个主题，可以通过在config.toml中通过参数theme指定主题，即theme目录下的子目录名字， 也可以在执行hugo命令渲染时通过增加flag参数–theme=xx指定。

  2. 下载主题地址：https://themes.gohugo.io/

4.使用hugo创建一篇文章

  1. 创键一篇文章命令：hugo new 文章的路径/文章的名称

    PS C:\Hugo\Sites\hugoShare> hugo new post/hugo-share.md
C:\Hugo\Sites\hugoShare\content\post\hugo-share.md created

  2. 新建的文档自带一段文件头（ front matter）：

    ---
title: "Hugo Share"
date: 2019-10-18T17:18:42+08:00
draft: true
---

    title：表示这个文档的标题；

    date：表示这个文档创建的时间，这个时间的显示格式，可以在config.toml里配置date_format参数；

    draft：表示这个文档是否是草稿，值为true是草稿，草稿的文档不会生成；

    除此之外，你还可以添加其他信息：

    author: "zzl" # 作者
tags: ["hugo"] # 所属标签
categories: ["share"] # 所属分类
menu: ""
banner: "banners/7148951717_9bbf185db3_h-1018x460.jpg" # banner图地址
slider: true # 是否显示在轮播图里，与config.toml文件里的[params.slider]参数配合使用
disable_profile: true # 是否显示侧边栏

    具体参见 https://gohugo.io/content-management/front-matter/ 和主题的使用说明

  3. 完善你的文章内容；

生成的命令就是：hugo
